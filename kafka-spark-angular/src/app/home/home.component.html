This project is a dynamic log management system that allows you to monitor and analyze logs and errors for different patterns (e.g., data-center configurations). It integrates with Elasticsearch for log storage and Kibana for visualizing dashboards. The key features include:

  • Dynamic configuration for different patterns (e.g., data-center-1, data-center-2).
  • Customizable log filters to extract meaningful data from log files.
  • Error filters to identify and track specific errors in the logs.
  • Integration with Kibana for real-time dashboards (errors and log dashboards).
  • Easy addition of log files for monitoring against predefined filters.

Below is a step-by-step guide to configure and use the system effectively:

  1. Add a Pattern: Start by adding a new pattern (e.g., data-center-1). Each pattern corresponds to a unique log source or configuration.
  2. Define Log Filters: Set up filters for logs (e.g., Date, Time, Level, Component, and Content) to extract specific information from your log files.
  3. Define Error Filters: Configure error filters to identify key error patterns (e.g., java\\.\\S+Exception, Call From ([a-zA-Z0-9.-]+)).
  4. Add a Log File: Upload a log file that corresponds to the defined pattern and filters. The system will automatically parse the file and identify errors and logs based on the configured filters.
  5. Configure Dashboards: Add Kibana dashboard links for both logs and errors visualization. Example:
    • Normal Dashboard: For logs visualization.
    • Errors Dashboard: For error monitoring.
  6. Monitor and Analyze: Use the embedded Kibana dashboards to analyze real-time log and error data, ensuring smooth operation and quick troubleshooting.

With this project, you can centralize and monitor logs for different data centers or systems dynamically, ensuring efficient log analysis and error tracking.
